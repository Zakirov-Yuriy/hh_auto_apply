# LinkedIn (Easy Apply)

Бот теперь умеет работать на двух площадках: **hh.ru** и **LinkedIn Jobs**.
Логика основного цикла общая, площадка выбирается переключателем.

## Как переключать площадку

Через флаг запуска:

```bash
python run.py --platform hh         # как раньше (по умолчанию)
python run.py --platform linkedin   # LinkedIn Easy Apply
```

Или через переменную окружения (в `.env`):

```env
PLATFORM=linkedin
```

Флаг `--platform` имеет приоритет над `PLATFORM`.

## Настройки LinkedIn (.env)

LinkedIn использует те же ключевые слова поиска, что и hh (`HH_SEARCH_QUERY`),
но локацию задаём отдельно текстом (числовые регионы hh там не работают):

```env
PLATFORM=linkedin
HH_SEARCH_QUERY=python developer
LINKEDIN_LOCATION=Remote        # например: Russia, Germany, Remote. Пусто = без фильтра
HH_REMOTE_ONLY=true             # добавит фильтр "удалённо"
HH_MAX_APPLIES=20               # начните с небольшого лимита
HH_MIN_SLEEP=5
HH_MAX_SLEEP=12
```

Сессии браузера у площадок раздельные:
- hh.ru — папка `.hh_user`
- LinkedIn — папка `.linkedin_user`

При первом запуске откроется окно браузера: войдите в LinkedIn вручную
(включая 2FA, если включена) и нажмите Enter в консоли. Дальше сессия
сохранится, повторный вход не потребуется.

## Что важно знать

- Автоматизируется **только Easy Apply** (отклик внутри LinkedIn). Вакансии с
  внешней формой (кнопка "Apply" ведёт на сайт компании) бот пропускает.
- Если форма Easy Apply содержит обязательные вопросы, которые бот не умеет
  заполнять, он аккуратно закрывает форму и переходит к следующей вакансии,
  ничего не отправляя.
- Сопроводительное письмо для LinkedIn необязательно. Если поле письма в форме
  есть, бот вставит текст из `data/cover_letter.txt`.

## Риски (читать обязательно)

LinkedIn в отличие от hh.ru активно борется с автоматизацией. Автоматические
отклики формально нарушают пользовательское соглашение LinkedIn, и есть
вероятность ограничения или блокировки аккаунта. Чтобы снизить риск:

- ставьте небольшой лимит откликов за запуск (`HH_MAX_APPLIES`, например 10–20);
- держите крупные паузы между действиями (`HH_MIN_SLEEP` / `HH_MAX_SLEEP`);
- не запускайте бота часто и помногу;
- не запускайте в headless без необходимости.

Селекторы LinkedIn (в `infrastructure/browser/linkedin_selectors.py`) могут
устаревать при смене вёрстки сайта. Если бот перестал находить кнопки или
карточки, правьте в первую очередь этот файл.
