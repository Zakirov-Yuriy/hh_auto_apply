# Функционал AI-Powered Custom Questions

## Статус: ✅ ГОТОВ К ИСПОЛЬЗОВАНИЮ

Бот теперь **автоматически обнаруживает и заполняет кастомные вопросы** на форме откликов HH.ru, используя AI для генерации ответов.

## Что реализовано

### 1️⃣ Обнаружение кастомных вопросов
- Автоматически находит textarea поля с именами типа `task_*_text`
- Пытается извлечь текст вопроса из labels и parent элементов
- Поддерживает различные HTML структуры форм

### 2️⃣ Генерация AI-ответов
- Использует модель **`openai/gpt-oss-120b:free`** через OpenRouter API
- Генерирует контекстные ответы на основе:
  - Данных резюме (должность, зарплата, опыт, навыки)
  - Названия вакансии
  - Текста самого вопроса
- Поддерживает ротацию API ключей

### 3️⃣ Автоматическое заполнение формы
- Заполняет textarea поля сгенерированными ответами
- Валидирует успешное заполнение
- Обрабатывает ошибки gracefully

### 4️⃣ Интеграция в основной процесс
- Автоматически вызывается при отклике на вакансию
- Запускается после открытия формы откклика
- Не блокирует стандартный процесс отклика

## Использование

### Конфигурация (.env)
```env
# Включить AI-powered ответы на кастомные вопросы
USE_AI_COVER_LETTER=true
AI_MODEL=openai/gpt-oss-120b:free
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxxx

# Опционально: несколько ключей для ротации
OPENROUTER_API_KEYS=key1,key2,key3
```

### Автоматическое использование
Функционал включается автоматически при наличии конфигурации. Просто запустите обычный процесс отклика:
```bash
python run.py
```

### Тестирование

## Как это работает изнутри

1. **Шаг 1**: После клика на "Откликнуться" форма открывается
2. **Шаг 2**: Бот сканирует форму на предмет textarea полей с именами `task_*_text`
3. **Шаг 3**: Для каждого найденного поля:
   - Извлекается текст вопроса
   - Извлекается контекст резюме
   - Отправляется запрос к OpenRouter API
   - Получается AI-сгенерированный ответ
   - Ответ вставляется в textarea
4. **Шаг 4**: Форма готова к отправке стандартным образом

## Ограничения

1. Поддерживаются только textarea поля (не select, checkbox, radio)
2. Качество ответов зависит от модели AI
3. Функционал работает только на вакансиях с HTML структурой, совместимой с парсером
4. Требуется активный OPENROUTER_API_KEY

## Troubleshooting

### Вопросы не обнаруживаются
- Проверьте наличие textarea полей с name типа `task_*_text`
- Откройте DevTools и найдите textarea элементы вручную
- Возможно, HTML структура нестандартная

### Ответы не генерируются
- Проверьте OPENROUTER_API_KEY в .env
- Проверьте баланс в OpenRouter
- Посмотрите debug логи для ошибок API

### Поля не заполняются
- Убедитесь, что textarea видимы на странице
- Проверьте консоль браузера на JavaScript ошибки
- Попробуйте вручную проверить, можно ли заполнить поле
